# Проект приложения трекера задач сотрудников
## Описание:
**Cерверное приложение для работы с базой данных, представляющее собой трекер задач сотрудников.**
## Цель проекта:
Реализовать приложение, которое позволяет эффективно управлять заданиями, назначенными сотрудникам, и обеспечивать прозрачность процессов выполнения задач
## Установка:
1. Клонируйте репозиторий:
```python
https://github.com/PetrMaev/task_tracker.git
```
```python
git@github.com:PetrMaev/task_tracker.git
```
2. Установите зависимости:
```python
poetry install
```
## Наполнение данными базы данных:
Для наполнения данными базы данных используйте фикстуры и кастомные команды.
1. Кастомная команда для заполнения базы данных задачами:
```python
python manage.py add_data
```
2. Кастомная команда для создания суперпользователя:
```python
python manage.py csu
```
3. Кастомная команда для заполнения базы данных пользователями для тестирования:
```python
python manage.py add_users
```
## Использование:
Для запуска программы введите в терминале следующую команду:
```python
python manage.py runserver
```
## Запуск приложения с использованием docker-compose
1. Для запуска проекта с docker-compose выполните следующую команду:
```python
docker-compose up --build
```
2. Для остановки работающих контейнеров используйте следующую команду:
```python
docker-compose down
```
4. Для просмотра логов и мониторинга работы контейнеров используйте команду:
```python
docker-compose logs
```
3. Для вывода списка всех контейнеров используйте команду:
```python
docker-compose ps
```
## В проекте реализованы:
1. Регистрация пользователей.
2. Авторизация.
3. Доступ к созданию задач у пользователей ограничен. Для создания и постановки задач сотрудникам пользователь должен иметь статус "руководитель".
4. Пользователи могу просматривать поставленные им задачи, сроки выполнения, связанные задачи без возможности редактирования.
5. Доступ редактирования задач открыт только для руководителей и только тех задач, авторами которых они являются.
6. Пагинация списка задач, пользователей, менее загруженных сотрудников.
7. Сортировка пользователей по количеству активных задач.
8. Возможность поиска наименее загруженных сотрудников.
## Дополнительная информация:
### I. Настройки удаленного сервера.
1. Подключитесь к виртуальной машине с помощью команды:
```python
ssh <имя_пользователя>@<IP_адрес сервера>
```
2. Установка Git

Откройте терминал на сервере и выполните команду:
```python
sudo apt update
sudo apt install git
```
Убедитесь, что Git установлен корректно, выполнив команду:
```python
git --version
```
Если всё установлено правильно, команда выведет номер версии Git.

3. Клонирование репозитория с GitHub

Перейдите в директорию, где вы хотите разместить код вашего приложения. Например:
```python
cd /var/www
```
Где /var/www — это директория, куда вы хотите склонировать проект.

Затем выполните команду для клонирования репозитория:
```python
git clone https://github.com/username/repository.git
```
Замените username/repository на путь к вашему репозиторию.

После выполнения этой команды весь код из репозитория будет скопирован на ваш сервер в указанную директорию.
4. Запуск контейнеров

Перейдите в директорию, где находится ваш файл docker-compose.yml. Например:
```python
cd /var/www/repository
```
Замените /var/www/repository на директорию, куда проект был склонирован.

Выполните команду для запуска контейнеров:
```python
docker-compose up -d
```
Эта команда соберет Docker-образы, если они еще не собраны, запустит контейнеры и свяжет их между собой 
в соответствии с конфигурацией, описанной в docker-compose.yml.
